Network Allowlist

If your organization uses secure web gateways (e.g., Cloudflare Zero Trust/Gateway) or content filters that block newly registered domains, please allow the hostnames below. DATAEX is a static vendor site (GitHub Pages over HTTPS) with no tracking and no user uploads.

Hostnames to Allow (by hostname, not IP)

  • www.dataexapp.com — canonical website (served by GitHub Pages via Fastly)
  • dataexapp.com — redirects (HTTP 301) to https://www.dataexapp.com/
  • Optional, if your policy blocks third-party CDNs or GitHub Releases:
  • cdn.tailwindcss.com — CSS framework loaded on the site
  • github.com — release landing pages
  • objects.githubusercontent.com — actual file host for GitHub Releases downloads

IP allowlisting is not recommended for GitHub Pages or Releases (anycast IPs change). Use hostnames.

What this site does

  • Static product pages (EULA, Privacy, About, Changelog)
  • Enforced HTTPS on www.dataexapp.com
  • Download link points to a signed Windows trial ZIP on GitHub Releases
  • No tracking pixels; no forms; no user data collection

Quick verification (IT can run)

Confirm DNS, HTTPS, and redirect behavior from any terminal:

# DNS: www CNAME points to GitHub Pages
nslookup -type=cname www.dataexapp.com 1.1.1.1
nslookup -type=cname www.dataexapp.com 8.8.8.8

# HTTPS on canonical host
curl -I https://www.dataexapp.com/

# Apex redirects to canonical
curl -I https://dataexapp.com/
curl -I http://dataexapp.com/

Expected: www.dataexapp.com → CNAME meataol.github.io; https://www.dataexapp.com/ returns 200; apex returns 301 to the canonical URL.

Cloudflare Zero Trust / Gateway (example policy)

If your organization uses Cloudflare Gateway, create an allow list and two simple policies:

  1. List: Hostname list allow-dataexapp containing dataexapp.com and www.dataexapp.com.
  2. DNS policy: If Domain in allow-dataexapp → Allow (place above “Newly Registered/Seen Domains” blocks).
  3. HTTP policy (if HTTP filtering is enabled): If Hostname in allow-dataexapp → Allow.
